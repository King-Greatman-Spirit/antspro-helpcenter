Assest General Equipment Request Slips Dashboard Help Center

This dashboard offers a centralized overview of General Equipment Request Slips across various workflow stages. Easily track equipment requests from submission through authorisation, approval, and final dispatch.

You Have Questions, We Have Answers

Navigating the Operations Dashboard

  • Dashboard: Click to return to the main dashboard overview.
  • Back: Go back to the previous page you visited.
  • Refresh: Reload the current page to view the latest updates.

Tabs Explained

Each tab on the page filters equipment request records based on their current workflow stage:

  1. Pending Authorisation: Contains requests awaiting authorisation from the designated authority before further processing.
  2. Pending Approval: Lists requests that have been submitted but are awaiting official approval.
  3. Pending Dispatch: Shows requests that have been approved and are awaiting dispatch.
  4. Dispatched: Displays requests that have been fully processed and dispatched.
  5. Pending Submission: Displays requests that have been prepared but not yet formally submitted.

Navigating and Using the Table Efficiently

To make the most of the Equipment Request Slips table, the platform provides multiple user-friendly tools to help you find, view, and analyze records quickly:

  1. Search Input: Located above the top-right corner of the table, this feature allows you to instantly filter results by typing keywords such as Reg. No., Project, Reg. By, or any relevant value. The results update in real time as you type.
  2. Records per Page: The default view shows 10 records per page with controls to navigate through pages.
  3. Sortable Columns: Each table header allows you to sort data in ascending or descending order.

Understanding the Table Summary

Each row in the table presents essential details of each equipment request:

  1. Date: Shows when the request was submitted (e.g., "2018-09-10").
  2. Reg. No.: A unique registration number for each request (e.g., "4520").
  3. Project: Details the project associated with the request (e.g., "Admin/jan-dec.2018", "24 Inch Yokri Fyip").
  4. Reg. By: The person who registered the request (e.g., "Christopher Onyeocha Josephine", "Ujah John Sunday").
  5. Authorisation, Approval, Dispatch: Indicates the current status (all currently showing as "Pending" in the example).
  6. Edit Option: Allows modifications to requests that are still pending processing.

Note: The example shows equipment requests for various projects including pipeline maintenance ("24 Inch Yokri Fyip"), administrative work ("Admin/jan-dec.2018"), and verification tasks ("Dig Verification").

Pro Tip: Use the search function to quickly find requests by project type (e.g., "Inch") or requester name when working with multiple equipment requests.
