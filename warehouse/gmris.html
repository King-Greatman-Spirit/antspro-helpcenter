Navigating and Using the Request/Issue Slip Tables

Dashboard Overview - Material Request/Issue Workflow

The dashboard is organized into tabs that show the current stage of each material request. Here’s a breakdown of each:

  • Global Material Request/Issue Slips: Overview of all requests in all statuses.
  • Completed: Fully approved and dispatched requests. No further action required.
  • Pending Dispatch: Approved requests awaiting materials delivery or physical dispatch.
  • Pending Approval: Requests that are yet to be verified and signed off by the approving officer.
  • Pending Authorisation: Requests awaiting managerial or departmental authorization.
  • Pending Submission: Draft or newly created requests not yet submitted for review.

Table Functional Features:

  • Search: Use the search box at the top right to quickly locate specific requests by date, request number, project, or requester.
  • Records per Page: Allows users to control how many request slips appear on a single page (e.g., 10, 25, 50).
  • Pagination Controls: Scroll between different pages of pending records at the bottom of the table.
  • Sortable Headers: Click on column headers to sort the data ascending/descending.

Table Columns Explained:

  1. Date: The submission or request date of the material slip.
  2. Req. No: The unique request number for each entry.
  3. Project: The associated project or job tag for the requested materials.
  4. Req. By: The name of the requester who submitted the form.
  5. Status: Shows the current progress of the request (usually "Pending").
  6. Authorisation: Displays whether managerial approval has been provided.
  7. Approval: Indicates if the request has been verified and signed off.
  8. Dispatch: Status of the material dispatch — whether fulfilled or not.
  9. Edit: Click this button to update, modify, or correct request details before final approval.

Each material request must go through the full cycle of Request → Authorisation → Approval → Dispatch. Keep an eye on pending stages.

Tip: Use filters and pagination wisely to avoid missing pending records. Editing before posting ensures accurate records!

Workflow Tips

  • Edit requests only in Pending Submission stage
  • Use View/Print for authorization/approval reference

Important Notes

  • All completed requests are automatically archived
  • Editing disabled after submission to maintain audit trail
