Cash Advance Request Form (BGT Technical Limited)

This form is designed for requesting operational cash advances within B.G. Technical Limited's Pipeline and Well Services division. Complete all required fields to initiate the approval workflow.

You Have Questions, We Have Answers

Navigating the Project Dashboard

  • Dashboard: Click this to return to the main dashboard view.
  • New Project: Use this button to create a new project entry.
  • Back: Returns you to the previous page you were viewing.
  • Refresh: Reloads the page to ensure you're seeing the most current data.

Form Header Information

  • Company Header: Identifies the form as belonging to B.G. Technical Limited's Pipeline and Well Services division.
  • Recipient Information:
    • To: Specifies the advance recipient (e.g., "Unye-Awaji Greatman")
    • Department: The operational department (e.g., "Technical")
  • Project Details:
    • PROJECT: The project name (e.g., "NUN RIVER 7")
    • Date: The request date in YYYY-MM-DD format

Expense Line Items

The expense table is where you detail all advance requirements:

  1. S/No: Sequential numbering for line items (1-10 available)
  2. CODE: Optional accounting/GL code for the expense (e.g., "001")
  3. PURPOSE: Detailed description of the expense (e.g., "Diesel 3,000.00 litres")
  4. AMOUNT - ₦: The naira amount requested (e.g., "796,000.00")
  5. REMARKS: Additional details or calculations (e.g., "Diesel for Operation @265.001 x 3,000.00")
  6. TOTAL =₦=: Automatically sums all line items (currently shows 0.00 when blank)

Financial Details

  1. Amount in Words: Required field that spells out the total amount in naira (not shown in current form)
  2. Account to Charge: The project account being charged (e.g., "NUN RIVER 7")

Pro Tip: Complete all line items with detailed descriptions to expedite approval. Include unit costs and quantities in remarks when applicable.

Approval Workflow

The form tracks the complete authorization chain:

  1. Sign:
    • Dropdown shows "Pending" status
    • Recipient's signature confirms request accuracy
    • Shows signer name and date when completed
  2. Pass:
    • Secondary verification by department
    • Shows "Pending" status until approved
    • Records approver name when completed
  3. Approval:
    • Final management authorization
    • Dropdown shows "Pending" status
    • Records approver name when completed
  4. Received By: Documents who physically received the funds (currently "Pending")

Form Submission

  1. Created By: Automatically records the form creator (e.g., "Unye-Awaji Greatman")
  2. Creation Date: Timestamps when the form was prepared (e.g., "02/05/2025 15:25:24")
  3. Save & Submit Button:
    • Validates all required fields before submission
    • Initiates the approval workflow process
    • Locks the form from further edits after submission

Important: Ensure all financial details are accurate before submission. Changes cannot be made after the form enters the approval workflow.
