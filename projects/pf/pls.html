Projects Budget Authorization Help Center

This section helps you track and authorize all Project Budget/Profit & Loss Statements through the approval workflow - from submission to final authorization and implementation.

Financial Workflow Management

Navigating the Budget Dashboard

  • Dashboard: Return to the main dashboard overview.
  • New P&L/Budget: Create new profit and Loss Budget Project.
  • Back: Go back to the previous page you visited.
  • Refresh: Update to see the latest authorization status changes.

Authorization Workflow Stages

Budget documents progress through these approval stages:

  1. Below 25% OMP/Field: Initial review stage for budget viability assessment.
  2. Pending Implementation/Approval: Awaiting final implementation confirmation after approval.
  3. Pending Mgr's Approval: Requires sign-off from the project manager.
  4. Pending Authorisation: Final stage requiring executive authorization.

Authorization Best Practices

  • Verify all financial data before submission
  • Check OMPFitel compliance at initial stage
  • Use Authorise button only after full review
  • Contact Finance for clarification on P&L items

Critical Reminders

  • Budgets below 25% OMPFitel require special justification
  • All authorizations are logged for audit purposes
  • No edits allowed after COO/HOO approval
