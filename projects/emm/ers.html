About Equipment Return Slips (ERS)

The ERS system ensures proper documentation and tracking of equipment being returned from project sites, maintenance, or temporary allocations. Key purposes include:

  • Asset Accountability: Maintains records of equipment condition upon return.
  • Workflow Compliance: Ensures all returns follow approval protocols..
  • Inventory Management: Updates stock levels after verification.

All returns must undergo verification before final posting to the system.

Workflow Status Guide

Request Statuses

  • Pending Verification/Posting: Equipment physically returned but awaiting system update (e.g., 102,550 items pending).
  • Pending Reception: Return logged but not yet physically received.
  • Pending Approval: Awaiting supervisor validation for completeness.
  • Pending Authorisation: Requires HOD sign-off before final processing.
  • Pending Submission: Draft requests not yet sent for review.

Approval Roles & Responsibilities

  • Requester:Initiates the return and provides equipment details.
  • Supervisor: Reviews condition and approves/denies return.
  • HOD: Grants final departmental authorization.
  • Logistics: Confirms transport and receipt.
  • Recipient: Inspects and validates equipment condition.

Damage protocol:

  1. Document condition with photos
  2. Flag in system as "Damaged"
  3. Initiate damage report
  4. Notify supervisor immediately
  5. Isolate equipment if hazardous
Do not post to inventory until damage assessment is complete.
