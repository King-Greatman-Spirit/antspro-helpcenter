Work Order Dashboard Help Center

Welcome to the BGT Work Order Dashboard (Form 202). This interface helps you manage, track, and review work order requests from initiation to approval. The tabs and table offer real-time access to workflow statuses and important metadata for each submission.

You Have Questions, We Have Answers

Navigating the Dashboard

  • Dashboard: Go back to the system’s main overview page.
  • New WO: Create a new Work Order using Form 202.
  • Back: Return to the previously accessed section or page.
  • Refresh: Reload current view to show updated records or status.

Tabs Breakdown

  1. Ongoing: Work orders actively being processed across departments.
  2. Pending SAP: Requests that are awaiting synchronization or updates from the SAP system.
  3. Pending Approval: Requests submitted but still waiting for final approval from a designated authority.
  4. Pending Initiator's Sign: Forms that require the original requester's acknowledgment or signature before proceeding.

Table Summary

Below the tabs, a dynamic table appears showing records based on the selected status tab. Here's what each column means:

  1. Date: The date the request was logged into the system.
  2. SR No.: The unique serial/reference number for each submitted form.
  3. Project Description: A short summary of the project or task to be executed.
  4. Req. By: Indicates the name or department of the requester.
  5. Authorisation: Shows whether the request has received budget/managerial clearance.
  6. Approval: Displays the status of final approval by the relevant authority.
  7. View: A button/link to view full details of the Work Order or Form 202 entry.

Warning Notice: If you see an error like: mysqli_fetch_assoc() expects parameter 1 to be mysqli_result, boolean given, it usually means the system could not fetch results – possibly due to no matching records or a database query issue.

Tip: "No data available in table" simply means that no work orders currently match the selected status or filter.
